# Skill Directory Structure Spec

This is the standard layout every skill in `aws-solution-skills/` MUST follow. It is the prerequisite for the three agents (Amazon Quick · Kiro · Claude Code) to consume the same body of knowledge.

## Required tree

```
<solution-name>-skill/
├── README.md                          ★ skill's own overview and usage
│
├── quick/
│   └── SKILL.md                       ★ Amazon Quick entry (with frontmatter)
│
├── kiro/
│   ├── steering.md                    ★ Kiro Steering Rules
│   └── specs/
│       └── generate-<name>.md         ★ Kiro Spec
│
├── claude-code/
│   ├── CLAUDE.md                      ★ Claude Code project guidance
│   └── commands/
│       └── generate-<name>.md         ★ /generate-<name> slash command
│
├── shared/                            ⭐ the actual knowledge — referenced by all three tools
│   ├── reference/
│   │   ├── architecture.md            Solution architecture + WHY
│   │   ├── decision-tree.md           Conditional selection logic
│   │   ├── aws-services.md            Service / model catalog
│   │   └── constraints.md             Known limits and gotchas
│   │
│   ├── patterns/
│   │   ├── cdk-stacks.md              Full CDK source
│   │   ├── lambda-handlers.md         Full Lambda source + pitfall guidance
│   │   ├── frontend-pages.md          React + Tailwind + shadcn/ui
│   │   └── <domain-specific>.md       (optional) etl-transforms, bedrock-prompts, er-strategies, ...
│   │
│   └── examples/
│       └── <industry>.md              At least 2-3 domain instantiations
│
└── evals/
    └── <domain>-scenario.md           Verification scenarios (at least one)
```

## File-by-file role

### `<skill>/README.md`
Skill-level overview and usage. Linked from the root `README.md`.

| Section | Contents |
|---|---|
| Trigger examples | "Build me a unified customer profile system" |
| Supported tools table | Quick / Kiro / Claude Code entry paths |
| Key outputs | Major files generated by the skill |
| MCP requirements | Which MCPs are recommended/required |

### `<skill>/quick/SKILL.md`
Amazon Quick entry file. **Frontmatter required**:

```yaml
---
name: <slug>
display_name: <Title>
description: >
  1-2 lines describing what this skill builds, shown in the Skill catalog UI
trigger: "Single natural-language trigger phrase"
icon: 🧬
inputs:
  - <input1>
  - <input2>
tools:
  - aws_knowledge_mcp_server__aws_search_documentation
  - aws_knowledge_mcp_server__aws_read_documentation
  - run_python
  - file_write
  - file_read
  - start_task
---
```

The body is the **Phase 1 Discovery → Phase 2 Design → Phase 3 Generate → Phase 4 Validate → Phase 5 Deploy** workflow.

### `<skill>/kiro/steering.md`
Kiro's always-on Steering Rules. Contains **identity + workflow + knowledge-source list**. Code patterns themselves stay in `shared/patterns/`.

### `<skill>/kiro/specs/generate-<name>.md`
A Kiro Spec mapping requirement collection → implementation steps. Includes Discovery questions, Design outputs, and the file inventory the agent will produce.

### `<skill>/claude-code/CLAUDE.md`
Project-root guidance copied into a Claude Code project. Includes a **Hard Constraints** list (one-liners describing must-not-violate pitfalls).

### `<skill>/claude-code/commands/generate-<name>.md`
The `/generate-<name>` slash command definition. Same workflow as `quick/SKILL.md` but more concise.

### `<skill>/shared/`
**All the real knowledge lives here.** Tool entry files reference these paths.

Detailed authoring rules: [`shared-knowledge-pattern.md`](./shared-knowledge-pattern.md).

### `<skill>/evals/`
Black-box scenario verification. Confirms each tool follows the workflow correctly.

## Naming conventions

- Skill folder: kebab-case + `-skill` suffix (e.g. `unified-customer-profile-skill`)
- Skill slug (frontmatter `name` field): kebab-case, without `-skill` suffix (e.g. `unified-customer-profile-builder`)
- Slash command / spec name: `generate-<short-name>` (e.g. `generate-ucp`)
- shared files: kebab-case `.md`

## Impact of missing pieces

| Missing | Tool-specific impact |
|---|---|
| `quick/SKILL.md` frontmatter | Amazon Quick import fails |
| `kiro/steering.md` | Kiro cannot pick it up as always-on context |
| `claude-code/CLAUDE.md` | Claude Code does not load project guidance |
| `shared/reference/` or `shared/patterns/` | Tool files contain broken references — agent hallucinates code |
| `evals/` | No regression check on changes |

## Auto-skeleton

Copy [`template/`](../template/) verbatim to start:

```bash
cp -r template <new-name>-skill
```

The `template/` is empty placeholders following this exact structure.
